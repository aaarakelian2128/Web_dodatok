## Тема: Розробка статичного інтерфейсу Web-додатка 

### Інформація про студента
* **ПІБ:** Аракельян Альбіна Андріївна
* **Група:** КВ-32

### Завдання до роботи 
Розробити статичні Web-сторінки додатку та перевірити коректне їх відображення на екранах різного розміру.

**TaskFlow** — це веб-додаток для персонального тайм-менеджменту, розроблений з акцентом на мобільний інтерфейс (Mobile First). Проєкт дозволяє користувачам реєструватися, керувати своїми справами у зручному воркспейсі та налаштовувати особистий профіль.

**Функціональні вимоги до інтерфейсу:**.
* Реєстрація користувача (поля: ім'я, email, стать, дата народження).
* Вхід до сайту (поля: email, пароль).
* Профіль користувача (дані у табличному вигляді).
* Головна сторінка (емблема та опис).
* Робоча сторінка додатка: панель додавання та перегляду завдань зі статусами.

# Лабораторна робота №2

### Інформація про студента
* **ПІБ:** Аракельян Альбіна Андріївна
* **Група:** КВ-32

### Завдання до роботи
* розробити функціональність для статичних сторінок Web-додатку першої лабораторної роботи із використанням шаблону MVC
### Посилання на звіт
* **Звіт у Google Drive:** https://docs.google.com/document/d/1tvOxUtPziW_j8CU6JRaI5ZplKeVrMiIo5ydbo_qD6b4/edit?usp=sharing

Ця версія проєкту є результатом глибокого рефакторингу попередніх напрацювань. Основною метою було розділення відповідальності між компонентами для полегшення підтримки коду:

*   **Model (Модель)**: Ізолює логіку роботи з даними та `localStorage`.
*   **View (Вигляд)**: Відповідає за рендеринг інтерфейсу та прослуховування подій користувача.
*   **Controller (Контролер)**: Виступає посередником, що синхронізує стан моделі з відображенням.
*   **App.js**: Централізована точка входу (Entry Point), яка ініціалізує потрібні контролери залежно від поточного URL.

##Функуціонал

### 1. Система автентифікації та профілю
*   **Реєстрація та вхід**: Повноцінна система авторизації з перевіркою пари **Email/Password**.
*   **Керування сесіями**: Збереження стану авторизованого користувача через **Web Storage API**.
*   **Динамічний інтерфейс**: Автоматична генерація ініціалів та відображення персональних даних для авторизованих користувачів.
*   **Гостьовий режим**: Обмеження доступу до функцій системи для неавторизованих відвідувачів.

### 2. Керування завданнями (Workspace)
*   **CRUD операції**: Динамічне створення та видалення завдань у реальному часі.
*   **Персистентність**: Автоматична синхронізація списку завдань із браузерним сховищем.
